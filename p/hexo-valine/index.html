Hexo | Valine评论

Created2022-11-13|Updated2022-12-05|技术
|Word count:1.6k|Reading time:6min|Post View:|Comments:

部署Valine

LeanCloud

在使用Valine之前，我们首先需要注册一个LeanCloud账户(建议使用国际版)，并创建一个App

由于一般个人博主的评论并不会太多，因此我们选择免费的“开发版”应用就可以了。

Appid & Appkey

创建好作为存储评论的应用数据库之后，

在 你的应用->设置->应用凭证 中记录好给出的 App ID 和 App Key

需要注意的是，必须先在 你的应用->设置->安全中心 将用于Valine的网站域名加入“安全域名”中

否则之后的Valine评论框将无法使用。

域名(必须)

在 你的应用->设置->安全中心 中根据提示分别绑定 api访问域名 和 云引擎、ClientEngine 域名

如果没有域名可以免费注册一个点此前往

修改Matery主题配置文件

填入Appid Appkey 以及Api访问域名

到这一步，Valine的基本设置就完成了，你可以在博客里测试评论，

安装Valine-admin

目的

给原生valine添加 邮件发送 评论管理 等功能

部署

进入 你的应用->云引擎->Web->设置

设置环境变量以及云引擎域名。

以下参数需要正确设置。

注: ADMIN_URL参数填写你绑定的 云引擎、ClientEngine 域名 用于评论后台管理

变量 示例 说明
SITE_NAME Deserts [必填]博客名称
SITE_URL https://blog.steven.tk [必填]你的博客首页地址
SMTP_SERVICE 163 邮件服务提供商,支持 QQ 163 126 Gmail 等,建议使用国内服务商,国外如outlook等可能出现连接超时
SMTP_USER xxxxxx@qq.com [必填]SMTP登录用户
SMTP_PASS ccxxxxxxxxch [必填]SMTP登录密码（QQ邮箱需要获取独立密码）
SENDER_NAME Steven [必填]发件人
SENDER_EMAIL xxxxxx@qq.com [必填]发件邮箱
ADMIN_URL https://xxx.xxx.xxx/ [建议]云引擎、ClientEngine 域名，用于自动唤醒和评论管理
BLOGGER_EMAIL xxxxx@gmail.com [可选]博主通知收件地址，默认使用SENDER_EMAIL
AKISMET_KEY xxxxxxxx [可选]Akismet Key 用于垃圾评论检测，设为MANUAL_REVIEW开启人工审核，留空不使用
MAIL_SUBJECT ${PARENT_NICK}，您在${SITE_NAME}上的评论收到了回复 评论被回复邮件标题
MAIL_TEMPLATE 见下文 评论被回复邮件模板
MAIL_SUBJECT_ADMIN ${SITE_NAME}上有新评论了 博客收到评论发送给站长的邮件标题
MAIL_TEMPLATE_ADMIN 见下文 博客收到评论发送给站长的邮件模板

MAIL_TEMPLATE

MAIL_TEMPLATE_ADMIN

如 SMTP_SERVICE 中没有你使用的邮件服务提供商,也可以进行自定义:

添加下列参数

SMTP_HOST : 邮件服务提供商 SMTP 地址，如 qq : smtp.qq.com，此项需要自行查询或询问其服务商。

SMTP_PORT : 邮件服务提供商 SMTP 端口, 此项需要自行查询或询问其服务商。

SMTP_SECURE : 是否启用加密, 默认为 true，一般不需要设置，如有特殊请自行配置。 此项需要自行查询或询问其服务商。

填写完后一定要 保存 !

切换到「云引擎-部署」，部署模式选择「部署项目-Git部署」，分支「master」，

手动部署目标环境为「生产环境」

在Leancloud云引擎设置界面，

填写代码库并保存，即可一键部署。

评论管理

首先需要设置管理员信息。

访问管理员注册页面https://云引擎域名/sign-up，

注册管理员登录信息

完成后登录，就可以看到评论列表啦

防止休眠

目前实现了两种云函数定时任务来解决云引擎休眠的问题：

1、自动唤醒，定时访问Web APP二级域名防止云引擎休眠；

2、每天定时检查过去24小时内漏发的邮件通知。

进入「云引擎-定时任务」，创建两个定时任务：

选择 self-wake 云函数，Cron表达式为 0 */30 0-16 * * ? ，表示每天早0点到晚16点每隔30分钟访问云引擎。

选择 resend-mails 云函数，Cron表达式为 0 0 0 * * ? ，表示每天0点检查过去24小时内漏发的通知邮件并补发。

至此，Valine Admin 已经可以正常工作。有问题欢迎在评论区提出！
